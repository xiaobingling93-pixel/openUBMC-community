# SIG中心

### SIG 介绍

SIG (Special Interest Group) 特别兴趣小组是openUBMC社区管理运营的单位。openUBMC社区内容按照不同的SIG来组织。openUBMC所有SIG 组均是开放的，欢迎任何人来参与贡献。

SIG是针对一个或多个技术主题而成立的。每一个SIG在openUBMC Gitcode上会拥有一个或多个Repository。SIG的主要职责是推动技术落地并负责SIG内软件包的发布。

SIG的组织信息在openUBMC community仓库对应SIG目录的README.md文件中。开发者可以查阅SIG信息、SIG中Repository列表与联系方式等。欢迎通过邮件列表与公开例会的方式参与SIG交流。

### SIG创建流程

申请创建SIG之前，请查阅现存的SIG列表并确认是否某个SIG已满足您的述求。正式申请创建SIG之前，建议在OpenUBMC论坛或公共邮箱发起讨论，这样可以为新SIG获得更多的关注度与反馈，有助于确认SIG的使命与范围并吸纳更多的SIG成员。

正式创建SIG需要参照SIG管理指南在community仓库创建Pull Request，并向社区技术委员会发邮件申请。技术委员会会定期组织评审决定是否通过申请。

### SIG角色说明

| 角色        | 主要职责范围                                               | 要求                                                   |
| ----------- | ---------------------------------------------------------- | ------------------------------------------------------ |
| Contributor | 项目贡献者，认领Isssue 和 PR 并在 PR 提交前完成测试        | GitCode注册成员                                        |
| Committer   | 项目代码审核者，分配Issue与PR，并审核代码的质量和正确性    | SIG 的积极贡献者，经验丰富，愿意投入精力参与到审核工作 |
| Maintainer  | 项目Owner，确定SIG所负责项目的技术路线并制定项目的发布计划 | 经验丰富，富有责任心、出色的技术能力和管理能力         |

### SIG List

sig-QA
The QA SIG aims to deliver openUBMC test harnness, test openUBMC software functionality, and improve the test capabilities.

sig-doc
The doc SIG aims to provide users with documentations and knowledge related to openUBMC, reduce the learning curve and facilitate the openUBMC utilization.

sig-infra
The infra SIG aims to build, manage and maintain the infrastructure of the openUBMC community, including the community official website, forum, code hosting and etc.
