# sig-security
sig-security在openUBMC社区有两项职责：
1. 通过接收和响应openUBMC安全问题报告、提供社区安全指导，开展安全治理等活动提升社区产品的安全性。
2. 看护与规划社区内安全相关组件架构、演进以及竞争力。

### 工作职责
**协助漏洞修复**：确保及时修复已知漏洞。审视社区发布范围内漏洞修补情况，提供安全的社区版本。
**响应安全问题**：响应上报的安全问题，跟踪安全问题的处理进展，并遵循安全问题披露策略对安全问题在社区内进行披露和公告。
**安全编码规则**：普及安全编码知识是安全团队的目标。安全团队会努力创建文档或开发工具来帮助开发团队避免软件开发过程中的常见陷阱。安全团队还会尝试回答在开发和使用过程中遇到的任何问题。
**参与代码审核**：安全团队希望能够通过代码审核帮助团队提前发现代码中的漏洞。

# 组织会议
sig-security每双周2开展例会。
**例行议题**包括：
社区漏洞处理进展跟踪
社区安全工具进展跟踪
其他社区安全专项议题，如安全指南、漏洞奖励计划等。

希望参与者可以订阅sig-security maillist
并在会议通知邮件的etherpad中添加与会人与议题信息，sig-security欢迎例会旁听、交流。
